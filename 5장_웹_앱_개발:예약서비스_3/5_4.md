# 상태유지기술(Cookie & Session) - BE

4.1 상태정보란?
~~~
HTTP : stateless, connectionless

- 이를 해결하기 위해 cookie 와 session을 사용해보자
~~~
cookie
~~~
- 사용자 컴퓨터에 정보를 저장

- 유효시간이 지나면 사라짐

- 다른사람이, 시스템이 볼 수 있음

쿠키의 동작
	1. 클라이언트가 서버에 요청을 한다.
	2. WAS가 쿠키를 생성해서 
	3. client에게 response로 쿠키를 보낸다.
	4. 그 이후 client는 쿠키를 request에 넣어서 보내준다. (WAS는 쿠키를 검사해서 이후 처리한다.)
~~~
session
~~~
- 서버에 정보를 저장

- 서버가 종료되거나 유효시간이 지나면 사라짐

세션의 동작
	1. 클라이언트가 서버에 요청을 한다.
	2. 서버는
		2.1 세션키를 생성한다.
		2.2 세션키를 이용한 저장소를 생성한다.
		2.3 세션키를 담은 세션ID를 생성한다.
	3. client에게 response로 세션ID를 보낸다.
	4. 그 이후 client는 세션ID를 request에 넣어서 보내준다. (WAS는 세션ID로 세션키를 얻어내 저장소를 찾아 들어있는 정보를 사용한다.)
~~~
4.2 쿠키란?
~~~
- 하나의 쿠키는 4K Byte 크기로 제한

- 브라우저는 웹 사이트 당 20개의 쿠키 허용

- 모든 웹 사이트를 합쳐 최대 300개 허용
~~~
